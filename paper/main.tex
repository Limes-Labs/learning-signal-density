\begin{abstract}
This draft studies whether apparent sample efficiency should be measured at the
level of a whole learning loop rather than at the level of a model update
alone. We define learning signal density as heldout quality improvement per
external observation and charged internal compute, then evaluate raw examples,
question-answer expansion, oracle counterfactual expansion, and train-only
ranked induction policies in a controlled causal-text domain. The present
evidence is intentionally narrow: deterministic CPU tiny-MLP artifacts with
fresh-seed confirmation, explicit heldout isolation, and generated tables
pulled from committed JSON. The strongest current positive result is not a
frontier-model claim. It is a reusable measurement protocol plus a concrete
finding: representation width and model profile can improve density, but
scarce-sample generated-label noise remains visible at 24 and 32 external
materials; even a non-deployable post-hoc policy envelope reaches the current
target only at 48 materials.
\end{abstract}

\paragraph{Keywords.}
sample efficiency; data transformation; synthetic data; counterfactual
generation; internal compute; learning-loop measurement; small neural audits.

\section{Introduction}

Token-count comparisons between biological learning and language-model
pretraining often treat data as if it arrived in a single undifferentiated
stream. That framing misses the mechanism this work wants to measure. A
learning system selects observations, rewrites them, asks questions, generates
counterfactuals, replays fragments, receives feedback, and spends internal
compute before any parameter update becomes useful.
Nearby mechanisms include token/data selection
\cite{lin2024rho1,li2024datacomp,xia2024less}, process feedback
\cite{lightman2023verify}, and replay \cite{schaul2015prioritized}. This draft
does not treat those literatures as interchangeable; it asks how to charge and
compare them inside one loop-level measurement protocol.

The research question is therefore not ``are language models or brains more
sample efficient?'' The sharper question is:
\emph{how much heldout improvement does a learning loop obtain per external
event once transformation, replay, feedback, selection, and internal compute
are charged?} This draft begins with a small but fully auditable causal domain
because the hidden rulebook, train/validation/heldout split, and generated-label
provenance can be controlled exactly.

\paragraph{Contributions in the current repository.}
The repository currently provides four pieces of evidence. First, it defines a
costed artifact schema for external events, internal examples, internal tokens,
charged transform cost, neural step count, estimated neural operations, and
claim scope. Second, it implements train-only and oracle transformation
policies under fixed heldout isolation. Third, it records fresh-seed tiny-MLP
sample-budget, feature-width, and epoch/width ablations. Fourth, the paper
tables below are generated from committed result JSON, so the manuscript cannot
silently drift away from the artifacts.

\paragraph{Non-claims.}
This is not a transformer-scale result, not a language-model leaderboard, and
not evidence that synthetic data is universally good. The current findings are
mechanism evidence for a measurement protocol. They justify more serious
replications; they do not replace them.

\section{Measurement Protocol}

Let a learning loop be
\[
S = (\theta, D_{\mathrm{ext}}, \pi_{\mathrm{select}},
\pi_{\mathrm{transform}}, \pi_{\mathrm{train}}, R, C).
\]
Here $\theta$ is the learner, $D_{\mathrm{ext}}$ is the externally observed
material, $\pi_{\mathrm{select}}$ chooses what to use, $\pi_{\mathrm{transform}}$
creates internal training examples, $\pi_{\mathrm{train}}$ updates the learner,
$R$ is the feedback or evaluation channel, and $C$ is charged internal compute.

For a heldout quality metric $Q$, external-event count $N_{\mathrm{ext}}$, and
charged internal compute $C_{\mathrm{int}}$, we track
\[
\mathrm{external\ sample\ efficiency} =
\frac{\Delta Q}{N_{\mathrm{ext}}},
\qquad
\mathrm{compute\ efficiency} =
\frac{\Delta Q}{C_{\mathrm{int}}},
\]
and
\[
\LSD =
\frac{\Delta Q}{N_{\mathrm{ext}} C_{\mathrm{int}}}.
\]
The reported signed version uses improvement over the majority baseline as
$\Delta Q$. Negative values remain negative instead of being clipped away.

\section{Anti-Cheat Rules}

All transformation and selection policies must be chosen without heldout
examples. Validation labels may be used only by conditions that declare and
charge validation use. Train-only ranked induction may use confidence, support,
and salience signals computed from training material, but it may not inspect
heldout accuracy. Oracle counterfactual expansion is allowed only as an upper
bound inside the synthetic world and must be labeled as oracle-generated.

Every committed result artifact carries a claim scope. The current neural
artifacts explicitly record that they are fresh-seed confirmations, use a
synthetic domain, do not use heldout data for selection, and are not yet
paper-ready claims. The generated paper-table script refuses artifacts that
violate those scope flags.

\section{Experimental Design}

The domain is a synthetic causal-text world with hidden rules and fixed splits
by material/stimulus pair. The learner observes textual or question-answer
views of a material, predicts heldout labels, and reports improvement over a
majority baseline. The current neural backend is a deterministic CPU MLP with
hashed features. It is intentionally dependency-light so that the artifact can
run in ordinary CI, but it is still a neural replication rather than the first
linear pilot.

The latest ablation path asks two practical efficiency questions. First, once
1024 hashed features improve the signal frontier, how much epoch/width budget
is actually needed? The tested profiles compare 32x8, 16x8, and 8x8 MLPs at
1024 features over 16, 24, 32, 48, and 64 external-material budgets. Second,
does explicit reliability selection repair the scarce-sample generated-label
failure? To test that, the 16x8 profile is rerun with a train-only agreement
gate, charged validation-ranked induction, and MDL rule expansion beside the
train-only ranked policies and oracle counterfactual headroom.
The latest selector extensions replace the expensive full validation portfolio
with a two-epoch linear proxy that scores the same candidate policies before
training only one final MLP, then test whether a raw-text abstention margin
reduces scarce-budget downside.

The current frontier has two useful signals and one important failure. First,
the 16x8 profile halves neural training operations relative to 32x8 for the
same feature width and improves the best self-ranked signed gain, so larger
training budgets are not automatically better. Second, reducing the profile to
8x8 halves operations again, but loses the self-ranked target and weakens the
main non-oracle result. Third, scarce-sample generated-label noise is not fixed
by shrinking the optimizer budget: all ranked 16x8 and 8x8 rows still fail at
32 materials, even when sample-aware ranking is positive at 16 and 24 for the
8x8 profile.

The reliability-policy rerun sharpens that failure mode. A train-only agreement
gate, which keeps only induced labels supported by agreeing rule projections,
does not solve the problem: it is worse than sample-aware ranking at 16
materials, tied with it at 32, and has much lower peak signed gain. Charged
validation-ranked induction slightly improves the 48-material point relative to
self-ranked induction, but it does not move the first target below 48 and it
does not repair the 32-material failure. MDL rule expansion is more conservative
at scarce budgets: it is positive at 16 materials and less negative than
self-ranked induction at 32 materials. However, it still remains below target at
32, reaches target only at 48, and has a lower peak signed gain than the
train-only ranked policies. This points to a more specific next step: a useful
generated-label policy must improve reliability at 24--32 materials without
paying so much selection cost that density collapses.

The post-hoc envelope in Table~\ref{tab:post-hoc-policy-envelope} is
deliberately not a deployable method: it chooses the best non-oracle condition
at each budget after seeing the completed heldout results. That diagnostic
still does not reach the signed-gain target below 48 materials. At 24 materials,
raw text is the best non-oracle row because every generated-label policy hurts
heldout accuracy. At 32 materials, MDL rule expansion is only the least bad
choice, not a positive result. The envelope switches again to validation-ranked
induction at 48 and self-ranked induction at 64. This makes the next research
problem sharper: a useful policy must learn when to abstain, when to compress
rules, and when to rank generated labels using only train signals or explicitly
charged validation signals.

Table~\ref{tab:validation-portfolio-selector} turns that diagnostic into three
deployable selector tests. The full portfolio trains six non-oracle candidate
MLPs, evaluates them on validation examples, chooses by validation improvement
over majority, charges the whole search, and then evaluates heldout once. The
linear proxy selector scores the same candidate policies with a two-epoch
linear classifier, charges those proxy fits, trains only the selected tiny MLP,
and then opens heldout once. The abstaining proxy adds a raw-text fallback: a
non-raw candidate must beat raw text by three validation examples before the
selector leaves raw text. The plain proxy improves the deployable selector
frontier, but it is still not a solved policy. At 48 materials it reaches 0.103
signed gain versus 0.076 for the full portfolio; at 64 materials it reaches
0.153 signed gain versus 0.109, while improving charged signed LSD from
0.000627 to 0.002091. The abstaining proxy improves downside control at scarce
budgets: it moves 16 materials from -0.105 to 0.000 signed gain and 24
materials from -0.041 to 0.006897. However, it still remains negative at 32
materials, lowers the 48-material gain to 0.082759, first reaches the target
only at 48, and still has lower 64-material density than fixed self-ranked and
sample-aware self-ranked induction. The result is therefore a real
selector-risk improvement, not a breakthrough on scarce-sample reliability.

\section{Discussion}

The mathematical shape suggested by the current evidence is a four-way
frontier. Feature width controls collision and representation capacity. Epoch
and hidden-unit budget control how much of that representation the learner can
use. Transformation policy controls whether generated labels add reliable
signal or amplify noise when external material is scarce. Selection policy
controls how much reliability is bought per extra unit of charged compute. A
useful next breakthrough is therefore unlikely to come from only increasing or
shrinking the MLP budget. The low-budget failure points instead toward
reliability-aware generated-label policies with sharper uncertainty estimates,
or a small transformer backend whose sequence structure can be charged honestly.
The post-hoc envelope also turns the problem into a selector-learning question:
future experiments should preregister train/validation features that predict
whether raw text, compact rules, validation-ranked candidates, or train-only
ranked candidates are appropriate before heldout evaluation is opened.
The selector probes are useful controls for that plan: the full portfolio shows
that trying every candidate on validation is too expensive, the linear proxy
shows that cheaper model hierarchies help, and the abstaining proxy shows that
simple validation-count caution can reduce downside at 16--24 materials without
repairing the 32-material failure.

The oracle counterfactual condition remains valuable because it estimates the
headroom available if generated examples were reliable. It should not be
confused with a deployable method. The non-oracle target is to close part of
the gap with train-only or explicitly charged validation mechanisms.

\section{Release Readiness}

This draft follows the release discipline used in Limes Labs public research
repos, but it is not ready for a versioned paper release. A release-quality
package should include a rendered PDF, source archive, result manifest,
checksums, artifact README, submission metadata, and the exact commands used to
rebuild the paper and reproduce the canonical artifacts. The current repo has
the result artifacts and generated table path; it still needs stronger model
replication and a frozen release bundle before public paper promotion.

\section{Limitations}

The current neural backend is a tiny MLP, not a transformer or language model.
The domain is synthetic and deliberately narrow. Oracle counterfactual labels
use the hidden world and exist only as an upper bound. The sample-aware and
self-ranked policies were developed through earlier discovery artifacts, so
fresh-seed confirmation helps but does not eliminate the need for independent
replication. Wall-clock time and hardware metadata are not yet sufficient for
a full compute-efficiency claim.
